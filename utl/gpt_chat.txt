# 🔥 Step 1 — Design a Proper Visualization Schema (THIS IS CRITICAL)

You want something like this:

```json
{
  "meta": {
    "title": "Loadest Container Problem",
    "type": "optimization"
  },
  "layout": {
    "canvas": "graph | grid | stack | free"
  },
  "components": [
    {
      "id": "container1",
      "type": "box",
      "x": 100,
      "y": 200,
      "width": 120,
      "height": 60,
      "label": "Weight: 10"
    }
  ],
  "timeline": [
    {
      "step": 0,
      "updates": [],
      "highlight": [],
      "explanation": "We start with all containers."
    },
    {
      "step": 1,
      "updates": [
        { "id": "container1", "color": "red" }
      ],
      "highlight": ["container1"],
      "explanation": "Container 1 is currently the heaviest."
    }
  ]
}
```

Notice:

* Components are defined once.
* Timeline only contains updates.
* Explanation per step.
* Code line per step (optional).

This avoids massive duplication.

## Color System Recommendation

Use semantic colors:

| Meaning  | Color   |
| -------- | ------- |
| Default  | #2D2D2D |
| Active   | #4CAF50 |
| Selected | #FFC107 |
| Warning  | #FF5722 |
| Info     | #29B6F6 |

Never let LLM choose arbitrary colors.

It should say:

```
"state": "active"
```

Frontend maps that to theme color.

# ✨ Step 4 — Animation Layer

This is what will make it feel “wow”.

Instead of re-rendering DOM each step:

Use:

* CSS transitions
* requestAnimationFrame
* Transform animations

Example:

```js
element.style.transition = "all 0.4s ease";
element.style.transform = "translate(...)";
```

For graphs:

Use smooth edge interpolation.

For stack:

Animate push/pop.

For array:

Pulse highlight.

# 🏛️ Step 6 — Better Layout Engine

Right now your layout is fixed grid.

You need:

### Option A — Absolute positioning (simpler)

LLM gives x,y coordinates.

### Option B — Auto-layout (better)

Frontend calculates layout:

* Graph → force-directed
* Tree → hierarchical
* Array → linear
* Stack → vertical

My recommendation:

Do NOT let LLM control pixel positions.

Let LLM define logical structure:

```
nodes
edges
levels
order
```

Frontend handles layout.

# 🎯 Example: Better BFS Representation

Instead of this:

```
node1 at x=100
node2 at x=200
```

LLM outputs:

```json
{
  "structure": {
    "type": "graph",
    "nodes": [
      {"id": "A"},
      {"id": "B"}
    ],
    "edges": [
      {"from": "A", "to": "B"}
    ]
  }
}
```

Frontend decides layout automatically.

This makes visuals consistent and beautiful.

# 🖥️ Should You Use Canvas or SVG?

Use:

Why?

* Perfect for graphs
* Easy to animate
* DOM-accessible
* Scalable
* Clean

Avoid raw Canvas unless performance becomes critical.
